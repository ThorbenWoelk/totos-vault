I was thinking about the predictions at the time concerning AI agents around 12 months ago:
Experts where waiting for the "App Store moment" of agentic AI. Envisioning a central platform where you can "get any AI agent you want for all of your needs". 

Doesn't feel as if that's what's happening any time soon. 

Instead, AI agents have been integrated into every existing and new app, tool, and SaaS. Yes, dedicated agents are a thing. But they're configured and harnessed by all the individual providers on THEIR platforms - with a varying degree of additional customizability by the user. 

There's simply no way you'd "download an agent" the same way for each of these use cases:
- coding locally
- customer service agent running in your web app
- an agent designing a prototype
- a personal agent building custom apps for you on-the-fly

Why is that?
- Different agent deliverables require dedicated presentation layers. Output can be as diverse as code, designs, texts, etc. Standard interfaces don't exist in all cases, yet. 
- Providers are incentivized to build their own ecosystems to lock you in as much as possible.
	- e.g. AWS DevOps agent *could* be a bedrock agent + skills + UI. But they chose to package as individual AWS service
- Humans still think in applications and product boundaries. This might change. 

It's still too early to see where this is going in the long run.
