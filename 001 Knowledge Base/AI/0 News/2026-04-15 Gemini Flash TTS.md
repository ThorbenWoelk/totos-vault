## Core idea

Google released Gemini 3.1 Flash TTS, a text-to-speech model focused on expressive, controllable speech generation. The key shift is that prompting starts to look less like plain narration and more like directing a voice performance: define the speaker, scene, delivery notes, transcript, and inline audio tags.

The API model name is `gemini-3.1-flash-tts-preview`.

## What changed

- **Audio tags**: Inline tags such as `[whisper]`, `[short pause]`, `[shouting]`, or `[yawn]` can steer delivery inside the transcript.
- **Natural-language direction**: Prompts can specify style, tone, accent, pace, breathing, articulation, and scene context.
- **Multi-speaker TTS**: The Gemini API supports single-speaker and multi-speaker audio; the documented multi-speaker setup currently configures up to 2 speakers.
- **Audio Profiles**: Google recommends defining character identity, archetype, scene, director's notes, sample context, and transcript so the model can maintain the intended performance.
- **Mid-transcript changes**: Inline tags let one speaker shift delivery without restarting the whole prompt or redefining the voice configuration.
- **Language coverage**: Google says Gemini 3.1 Flash TTS supports expressive control across 70+ languages.
- **Workflow export**: Google AI Studio can be used to test voices and export the configured parameters as Gemini API code.
- **Workspace rollout**: Google Vids now gets 30 conversational voice options powered by Gemini 3.1 Flash TTS.

## Benchmark signal

Google reports an Elo score of 1,211 for Gemini 3.1 Flash TTS on the Artificial Analysis TTS leaderboard, which is based on blind human preference comparisons. Google also says Artificial Analysis places the model in its "most attractive quadrant" for quality and cost.

This is useful as a market signal, not a complete evaluation. For production decisions I would still test:

- voice consistency across many generations
- prompt sensitivity and tag reliability
- accent quality in the target language
- latency and cost under real workloads
- failure cases for emotionally loaded or safety-sensitive text

## How to use it

The practical workflow is:

1. Draft the speech as a structured prompt with Audio Profile, Scene, Director's Notes, sample context, and transcript.
2. Use inline audio tags where the performance needs local control.
3. Test the output in Google AI Studio and the Voice Library.
4. Export the setup as Gemini API code.
5. Run production workloads through the Gemini API or Vertex AI preview, depending on deployment context.

## Safety and traceability

Google says all audio generated by Gemini 3.1 Flash TTS is watermarked with SynthID. The watermark is intended to be imperceptible during playback while still allowing detection systems to identify AI-generated speech.

The strategic point is that watermarking is built into the generated audio path rather than treated as an optional post-processing step.

## Current constraints

- Gemini TTS accepts text-only input and produces audio-only output.
- TTS sessions have a 32k token context window.
- TTS does not support streaming.
- The Gemini 3.1 Flash TTS API model is in preview.
- The documented multi-speaker configuration supports up to 2 speakers.

## Why it matters

This makes TTS more programmable. Instead of generating a neutral voice and fixing performance details downstream, teams can move more creative direction into the prompt and keep it alongside the text.

Good fit:

- AI voiceovers
- audiobooks and narration
- podcast-style generated dialogue
- localized product explainers
- character voices in learning tools
- scripted customer-support or onboarding audio

Less obvious fit:

- low-latency conversational voice agents, where the Live API may be the better fit
- applications needing streaming TTS
- workflows needing more than two distinct generated speakers in one API call
- regulated use cases where SynthID detection is not enough governance on its own
